SMASH — Landing Page Story Frames (two ICPs, one tool)
=======================================================
6 full-page frames, 1600x1000, fonts + logo embedded (render anywhere).
Each frame: eyebrow + Barlow headline (the message), an abstracted
Gmail + SMASH sidebar product shot, and an iOS-style dark ACTION CALLOUT
at the bottom (numbered lime badge + verb + sub-line + connector arrow)
with a subtle lime glow behind it.

THE STRATEGY — both ends of one transaction, both run SMASH:
  Story A = the tradie SENDING (customer enquiry -> priced quote)
  Story B = the supplier RECEIVING (tradie's request -> SKU-matched quote)
  The request A sends out is the request B receives in. Same tool, both sides.

STORY A — story_A_tradie/  (customer enquiry -> priced quote)
  A1_request_lands   "The job lands in your inbox"      callout 1 · THE REQUEST
  A2_reads_prices    "It reads it. It prices it."        callout 2 · SCAN EMAIL
  A3_quote_done      "Quote back in under a minute"      callout 3 · REPLY

STORY B — story_B_supplier/  (tradie request -> SKU match)
  B1_request_arrives "A tradie wants a price"            callout 1 · LANDS IN INBOX
  B2_sku_match       "It matches every line. No typing." callout 2 · SKU MATCH
  B3_quote_back      "Sent back. Into Xero. Paid."       callout 3 · SEND & SYNC

CALLOUT = the connector device. Numbered badge implies sequence; the lime
arrow implies flow to the next frame (steps 3 have no arrow — end of story).
Built to match the iOS App Store callout language so it's cohesive.

TOKENS  navy #0F172A · accent #DFFF00 · green #1FB873 · surface #F4F6F9
        line #E2E8F0 · muted #64748B · callout dark #0A1119
TYPE    Barlow Condensed Black Italic (headlines/labels/callout verb),
        Plus Jakarta Sans (body, UI, sub-lines)

ABSTRACTION NOTE: product is intentionally simplified — recognisable
Gmail+sidebar, a few clean lines, not the dense real screen. Resembles
how it works without overcomplicating. Edit copy/lines in the HTML;
